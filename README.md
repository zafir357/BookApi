# BookApi — REST API with Many-to-Many & DTO Pattern

A clean REST API built with **ASP.NET Core**, **Entity Framework Core**, and **AutoMapper** demonstrating a many-to-many relationship between `Books` and `Authors`, with a full DTO layer separating the API contract from the database model.

---

## Stack

- .NET 10 / ASP.NET Core
- Entity Framework Core + SQLite
- AutoMapper
- Swagger UI

---

## Database Schema

```
Books                    BookAuthor (join)         Authors
─────────────────        ─────────────────         ─────────────────
Id         int PK        BookId    int FK PK        Id         int PK
Title      string        AuthorId  int FK PK        Name       string
Year       int                                      Country    string
```

A book can have many authors. An author can have many books. The `BookAuthor` table is the explicit join entity with a composite primary key.

---

## DTO → Model → DTO Flow

The API never exposes raw database entities. Every request and response goes through a DTO layer.

### Why DTOs?

| Problem without DTO | How DTO solves it |
|---|---|
| Client can forge `Id` and overwrite any record | `Id` is simply not on the DTO — it can never arrive |
| `Book → BookAuthors → Book` circular reference crashes the JSON serializer | DTO breaks the cycle with a flat `List<AuthorSummaryDto>` |
| DB schema leaks to the client | DTO exposes only what the client needs |
| Renaming a DB column breaks the API | Model and DTO evolve independently |
| Validation mixed into domain model | `[Required]`, `[MaxLength]` live on the DTO only |

### POST — DTO → Model (create)

```
CreateBookDto                    Book (entity)
─────────────────                ─────────────────
string Title        ──────────►  string Title
int Year            ──────────►  int Year
List<int> AuthorIds ──────────►  ICollection<BookAuthor>
                                 int Id  ◄── generated by DB
```

```csharp
var book = mapper.Map<Book>(dto);
// AuthorIds wired separately in controller
db.Books.Add(book);
await db.SaveChangesAsync();
```

---

### GET — Model → DTO (read)

```
Book (entity)                    BookDto
─────────────────                ─────────────────
int Id              ──────────►  int Id
string Title        ──────────►  string Title
int Year            ──────────►  int Year
ICollection                      List<AuthorSummaryDto>
  <BookAuthor>                     └─ int Id
  └─ Author         ──────────►     └─ string Name
     └─ Id, Name
```

```csharp
var books = await db.Books
    .Include(b => b.BookAuthors)
    .ThenInclude(ba => ba.Author)
    .ToListAsync();

return Ok(mapper.Map<IEnumerable<BookDto>>(books));
```

EF Core loads the full 3-level graph. AutoMapper flattens it into a clean response. The circular reference never reaches the serializer.

---

### PUT — DTO → tracked Model (update)

```
UpdateBookDto                    Book (tracked by EF Core)
─────────────────                ─────────────────
string Title        ──────────►  string Title  ← updated
int Year            ──────────►  int Year      ← updated
                                 int Id        ← untouched
                                 BookAuthors   ← untouched
```

```csharp
var book = await db.Books.FindAsync(id);
mapper.Map(dto, book); // 2-arg: mutates tracked entity
await db.SaveChangesAsync(); // fires UPDATE SQL
```

The two-argument `mapper.Map(dto, book)` mutates the existing tracked entity in place. EF Core detects the changes and generates a targeted `UPDATE` statement.

## AutoMapper Profile

```csharp
// Book → BookDto (flatten many-to-many)
CreateMap<Book, BookDto>()
    .ForMember(dest => dest.Authors,
        opt => opt.MapFrom(src =>
            src.BookAuthors.Select(ba => ba.Author)));

// Author → AuthorSummaryDto (auto — same property names)
CreateMap<Author, AuthorSummaryDto>();

// Author → AuthorDto (flatten book titles)
CreateMap<Author, AuthorDto>()
    .ForMember(dest => dest.BookTitles,
        opt => opt.MapFrom(src =>
            src.BookAuthors.Select(ba => ba.Book.Title)));

// DTOs → Models (for write operations)
CreateMap<CreateBookDto, Book>();
CreateMap<UpdateBookDto, Book>();
CreateMap<CreateAuthorDto, Author>();
```

## Endpoints

### Books

| Method | Route | Description |
|---|---|---|
| GET | `/api/books` | Get all books with authors |
| GET | `/api/books/{id}` | Get one book with authors |
| GET | `/api/books/by-author/{authorId}` | Get all books by a specific author |
| POST | `/api/books` | Create a book (with one or many authors) |
| PUT | `/api/books/{id}` | Update title and year |
| DELETE | `/api/books/{id}` | Delete a book |

### Authors

| Method | Route | Description |
|---|---|---|
| GET | `/api/authors` | Get all authors with their book titles |
| GET | `/api/authors/{id}` | Get one author with their book titles |
| POST | `/api/authors` | Create an author |
| DELETE | `/api/authors/{id}` | Delete an author |

## Running Locally

```bash
# Install dependencies
dotnet restore

# Apply migrations and seed data
dotnet ef database update

# Run the API
dotnet run
```

Open Swagger UI at `http://localhost:5280/swagger`
